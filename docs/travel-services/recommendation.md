# 出張手配サービス導入 推奨案

**作成日**: 2026-04-01
**作成者**: 管理本部
**対象**: フューチャースタンダード 全社
**ステータス**: ドラフト

---

## エグゼクティブサマリー

出張手配の法人契約先として、**案4: じゃらんコーポレートサービス（JCS）+ ANA Biz / JALオンライン**を推奨する。

宿泊はJCSの法人一括決済、航空券は航空会社直接予約とすることで、**API連携の将来性・変更柔軟性・運用シンプルさ**の3点を最適化できる。

---

## 1. 推奨案の構成

| 手配対象 | サービス | 決済方法 | 備考 |
|---------|---------|---------|------|
| **宿泊** | じゃらんJCS | 法人一括決済 | 管理画面で上限金額・金券除外等のルール設定 |
| **航空券** | ANA Biz / JALオンライン | MFカード（法人カード利用モデル） | 会社用個人アカウント。MFカード明細で自動取込 |
| **新幹線** | スマートEX | MFカード | 会社メールで個人アカウント作成。MFカード明細で自動取込 |
| **レンタカー** | 各社直接予約 | MFカード | 会社メールでアカウント作成。MFカード明細で自動取込 |

---

## 2. 推奨理由

### 2.1 なぜJCS（宿泊）か

| 観点 | JCS | Racco | 評価 |
|------|-----|-------|------|
| 審査 | リクルート所定の審査あり（期間未確認） | 与信審査2週間+ANA Biz別途1.5ヶ月 | **要確認** |
| API連携 | 予約照会APIあり（NDA後に詳細取得） | おそらく不可（確認中） | **JCS優位** |
| 退職時対応 | 個人リクルートID（影響小） | 法人PW変更が必要 | **JCS優位** |
| 管理ルール | 上限金額・金券除外等を設定可 | 利用状況リアルタイム把握 | 同等 |
| 宿泊施設数 | じゃらんの豊富な在庫 | 楽天トラベルの在庫 | 同等 |

### 2.2 なぜ航空券は航空会社直接か

- **変更・キャンセルが頻繁に発生**する出張利用では、代理店を挟むと対応が遅れる
- ANA Biz / JALオンラインは法人割引＋直接変更が可能
- Raccoの航空券はパッケージ商品扱いのため、単独変更に制約がある

### 2.3 他の案を採用しない理由

| 案 | 不採用理由 |
|----|-----------|
| 案1（JCS単独） | 本案と実質同じ（案4として整理） |
| 案2（Racco単独） | 与信審査に時間、API連携困難、退職時運用が煩雑 |
| 案3（JCS+Racco両方） | 2社分の契約管理が煩雑、月50件規模では過剰 |

## 3. アカウント・決済の運用設計

### 基本方針

各サービスで**会社用の個人アカウント**（会社メールで作成）を社員ごとに用意し、MFビジネスカード（VISA）を決済手段に紐付ける。これにより：
- MF会計Plusでカード明細として自動取込 → 経費精算の手間を最小化
- 個人立替が発生しない
- 退職時はアカウント無効化で完結

### 3.1 新幹線: スマートEX

| 項目 | 内容 |
|------|------|
| **アカウント** | 会社メールで個人アカウントを作成（法人プランなし） |
| **決済** | MFビジネスカード（VISA）を登録 → カード明細でMF自動取込 |
| **費用** | 年会費無料 |
| **審査** | なし（即日利用可能） |
| **領収書** | Web/アプリから取得可能（15ヶ月間） |
| **IC乗車** | 交通系ICカードを紐付けてチケットレス乗車 |
| **注意** | 法人管理機能なし。利用実績は各自のアカウント or MFカード明細で把握 |

> **補足: エクスプレス予約（法人サービス）との比較**
> - 年会費¥1,100/枚だが割引額が大きい（スマートEXは片道数百円、EX予約は最大数千円）
> - 法人一括精算・利用実績データの一括ダウンロードが可能
> - 出張頻度が高い場合（月10件超）は導入検討の価値あり
> - ※EX-ICカードは2026年8月新規発行終了、2027年夏に乗車利用終了予定

### 3.2 航空券: ANA Biz

| 項目 | 内容 |
|------|------|
| **アカウント** | 法人契約 → 管理者が個人ユーザーを登録。部署・支店別に組織ID設定可 |
| **決済** | 「法人カード利用モデル」でMFビジネスカード（VISA）を設定 ※対応カード会社一覧で要確認 |
| **費用** | 初期費用・月額・年会費すべて無料 |
| **審査** | 法人カード利用モデルはANA側の審査は軽い（JCBカードレス決済の場合はJCB与信審査あり） |
| **マイル** | 搭乗者個人に付与（法人付与は不可、航空会社規約） |
| **法人割引** | ANA Biz割（通常運賃から一定率割引）あり |

### 3.3 航空券: JALオンライン

| 項目 | 内容 |
|------|------|
| **アカウント** | 法人契約（企業全体利用）→ 管理者＋個人ユーザー |
| **決済** | 「個人クレジットカード精算タイプ」or「法人クレジットカード精算タイプ」でVISA利用可 |
| **費用** | 初期費用・利用料すべて無料 |
| **審査** | JALによる取引審査あり（申込から利用開始まで2〜3週間） |
| **マイル** | 搭乗者個人に付与（ANA同様） |
| **精算タイプ** | 4種類: JAL精算（月締一括）/ 法人カード / 個人カード / 旅行会社精算 |

### 3.4 レンタカー

| 項目 | 内容 |
|------|------|
| **アカウント** | 各社サイトで会社メールのアカウントを作成 |
| **決済** | MFビジネスカード（VISA）で現地決済 or 事前決済 |
| **備考** | 法人契約は出張頻度に応じて検討（トヨタレンタカー法人会員等） |

### 3.5 アカウント運用まとめ

```
【入社時: 管理本部が実施】
1. スマートEX — 会社メールでアカウント作成、MFカード登録、交通系IC紐付け
2. ANA Biz — 管理画面から個人ユーザー追加、MFカード設定
3. JALオンライン — 管理画面から個人ユーザー追加、MFカード設定
4. じゃらんJCS — 会社メールでリクルートID作成、法人一括決済設定

【退職時: 管理本部が実施】
1. ANA Biz / JALオンライン — 管理画面からユーザー無効化
2. じゃらんJCS — 個人リクルートIDのため影響小（法人一括決済のみ解除）
3. スマートEX — MFカード解除を依頼（アカウント自体は個人所有）
```

## 4. 運用フロー

```
【出張申請〜精算の全体フロー】

① 事前承認
   社員: Slack /trip コマンドで出張申請
   上長: Slack上で承認

② 手配（各サービスの会社用個人アカウント + MFカードで決済）
   宿泊: じゃらん/楽天トラベル等で予約（MFカード決済）
         → PJ番号は①の /trip 申請時にHubSpot案件番号で紐付け済み
   航空券: ANA / JAL公式サイトで予約（MFカード決済）
   新幹線: スマートEXで予約（MFカード決済）
   レンタカー: 各社サイトで予約（MFカード決済）

③ 出張実施

④ 月次精算（管理本部）
   JCS: 翌月8日の確定CSVダウンロード → MF経費に一括登録
   ANA Biz/JAL: MFカード明細で自動取込（法人カード利用モデル）
   新幹線・レンタカー: MFカード明細で自動取込
   紙請求書（JCS）: スキャン → MF会計Plusに証憑添付

⑤ 仕訳
   購買管理システムの仕訳管理画面（/admin/journals）で一括登録
```

## 4. システム連携の将来像

```
【Phase 1: 導入時（手動運用）】
JCS CSV → 手動加工 → MF経費一括登録
ANA Biz/JAL → 請求書手動登録

【Phase 2: API連携（NDA締結後）】
JCS 予約照会API → 購買管理システムで自動取込
  → Slack通知（予約完了・変更・キャンセル）
  → MF経費への自動登録
  → PJ番号による部門別集計の自動化

【Phase 3: 統合管理（将来構想）】
全出張データを購買管理システムに集約
  → 出張費の予実管理ダッシュボード
  → 部門別・PJ別の出張コスト可視化
```

JCSの予約照会APIは、Phase 2以降のシステム連携において重要な差別化要素となる。Raccoにはこの選択肢がない。

## 7. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| JCSの請求書が紙郵送のみ | 経理の手間増 | スキャン→MF会計Plus添付で対応。Phase 2でデータ連携すれば請求書確認の手間は軽減 |
| 手配先が分散（JCS/ANA Biz/JAL/スマートEX） | 社員の手間 | 利用ガイドで手配先一覧を明示。Slack /trip で案内表示 |
| JCS予約照会APIの仕様が想定と異なる | Phase 2遅延 | NDA締結を早期に実施し、仕様確認後に開発計画を調整 |
| 楽天RaccoのAPI回答が好条件だった場合 | 判断見直し | 宿泊のみJCS、航空券+レンタカーをRaccoにする案3への切替を検討 |
