# 出張手配サービス比較検討メモ

**作成日**: 2026-04-01
**検討対象**: じゃらんコーポレートサービス (JCS) vs 楽天トラベル法人 (Racco)
**目的**: フューチャースタンダードの出張手配の法人契約先を決定

---

## 1. サービス比較表

| 項目 | じゃらんJCS | 楽天Racco |
|------|-----------|-----------|
| **宿泊予約** | 可 | 可 |
| **航空券予約** | 不可 | 可（TypeA契約でパッケージ商品） |
| **新幹線** | 不可 | 不可（TypeA） |
| **レンタカー** | 不可 | 可 |
| **請求書** | 紙ベース郵送（データ不可） | 1枚にまとめ可能 |
| **支払方法** | 法人一括決済のみ表示可 | 請求元により支払いルールが変わる |
| **承認機能** | なし | なし |
| **審査** | あり（リクルートによる所定の審査、期間不明） | あり（与信審査2週間程度、3パターン） |
| **ポイント** | 個人に付与（会社メールで新規作成で回避可） | 予約ユーザーIDに付与 |
| **管理画面ルール** | 金券なし・上限金額等を設定可 | リアルタイム利用状況把握可 |
| **退職時対応** | 個人リクルートID使用（影響小） | 法人アカウントのPW変更が必要 |

## 2. システム連携（MF会計Plus / 購買管理システム）

| 項目 | じゃらんJCS | 楽天Racco |
|------|-----------|-----------|
| **MFクラウド経費連携** | 不可（直接連携なし） | 不可（直接連携なし） |
| **MF会計Plus連携** | 不可 | 不可 |
| **API** | 「予約照会API」あり（NDA締結後に詳細提供） | おそらく不可（持ち帰り確認中） |
| **CSV出力** | 可（予約データ・確定データ） | 可（推定） |
| **データ確定タイミング** | リアルタイム（予約時）+ 確定（翌月8日） | リアルタイム利用状況あり |
| **SAP Concur連携** | 可（唯一の公式連携先） | 不明 |
| **間接連携の可能性** | JCS → ビズバンス（JTB） → MF経費 | 不明 |

### じゃらんJCSのデータ連携まとめ（小貫氏回答 3/24）

1. **予約照会API**: あり。企業側での開発が必要。NDA締結後に詳細情報提供可能
2. **MFとの直接連携**: 予定なし（SAP Concur以外とは連携しない方針）
3. **間接連携**: JCS → ビズバンス出張予約（JTBビジネストラベルソリューションズ） → MF経費 が可能な可能性あり
4. **提携カード**: 三井住友、三菱UFJニコス、JCB、AMEX、バクラクビジネスカード（MFカードなし）

## 4. 運用フロー想定

### 現行の購買管理システムとの統合

```
【じゃらんJCS利用の場合】
社員: /trip → Slackで事前承認
  ↓
社員: じゃらんで予約（法人一括決済）
  ↓ 予約時にPJ番号を質問項目に入力
管理本部: 月次でCSVダウンロード（翌月8日の確定データ）
  ↓
管理本部: CSVを加工 → /api/trip/import-csv でMF経費に一括登録
  ↓
管理本部: 紙の請求書をスキャン → MF会計Plusに添付

【楽天Racco利用の場合】
社員: /trip → Slackで事前承認
  ↓
社員: Raccoで宿泊+航空券+レンタカーを一括予約
  ↓
管理本部: CSVダウンロード → MF経費に一括登録
  ↓
管理本部: 請求書（1枚にまとまる）→ MF会計Plusに添付
```

## 5. 論点整理

### じゃらんJCSの強み
- 予約照会APIがある（NDA後に詳細取得可能 → 将来の自動化の余地）
- じゃらんの豊富な宿泊施設数
- 審査はあるが、Raccoの与信審査（2週間+ANA Biz 1.5ヶ月）より軽い可能性（要確認）
- 個人リクルートID利用のため退職時の影響が小さい
- 予約時にPJ番号等のカスタム質問項目を設定可能
- ビズバンス経由でMF経費連携の可能性あり

### 楽天Raccoの強み
- 航空券・レンタカーも一括予約可能（ワンストップ）
- 請求書が1枚にまとまる
- リアルタイム利用状況把握

### じゃらんJCSの弱み
- 宿泊のみ（航空券・レンタカーは別手配が必要）
- 請求書が紙郵送のみ
- MFカードとの連携なし

### 楽天Raccoの弱み
- 与信審査が必要（2週間+ANA Bizは1.5ヶ月）
- 退職時に法人アカウントPW変更が必要
- API連携はおそらく不可
- 新幹線はTypeA契約では不可
- 上長承認機能がない（JCSも同様）

## 6. 検討の方向性

### 案1: じゃらんJCS単独
- 宿泊はJCS、航空券はANA Biz/JALオンラインで個別手配
- 新幹線はスマートEXで個別手配
- レンタカーはMFカードで個別手配
- **メリット**: 導入が早い、APIの将来性
- **デメリット**: 手配先が分散、請求書が複数

### 案2: 楽天Racco単独
- 宿泊+航空券+レンタカーをRaccoで一括
- 新幹線はスマートEXで個別手配
- **メリット**: ワンストップ、請求書集約
- **デメリット**: 与信審査に時間、API連携困難

### 案3: 両方導入
- 宿泊: じゃらんJCS（宿泊施設数の強み）
- 航空券+レンタカー: 楽天Racco（ワンストップ）
- **メリット**: それぞれの強みを活用
- **デメリット**: 管理が複雑、2社分の契約管理

### 案4: じゃらんJCS + ANA Biz/JALオンライン（現行設計）
- 現在の設計書通り。JCSで宿泊、航空券は航空会社直
- レンタカー・新幹線はMFカードで個別
- **メリット**: 航空券は変更が多いので代理店を挟まない方が利便性高い（MTGメモ）
- **デメリット**: 手配先が分散
