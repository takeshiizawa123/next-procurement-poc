## 1. 基本方針

### 1.1 経路の使い分け

```
支払方法で自動分岐:

  会社カード → 案G: Google Drive証憑 + MF会計Plus API仕訳
  請求書払い → 案G: Google Drive証憑 + MF会計Plus API仕訳
  従業員立替 → 案C: MF経費で立替精算
```

### 1.2 カード利用の全量管理

```
従業員カード（1人1枚）:
  全利用を購買システムで管理
  ├── 購買 → /purchase → PO番号
  ├── 出張 → /trip    → TR番号
  └── 上記以外 → ★未申請アラート（統制強化）

管理本部カードA（購買用）:
  └── 購買 → /purchase → PO番号

管理本部カードB（サブスク用）:
  └── 固定仕訳ルール（MF会計Plusの自動仕訳で処理）
```

### 1.3 MF経費の役割

```
MF経費の用途:
  ○ 従業員立替の精算（振込データ生成）← 唯一の用途
  × 購買の証憑管理
  × 出張旅費の申請
  × カード利用の管理
```

## 2. 勘定科目・補助科目の体系

### MF会計Plusに必要な科目

```
【費用科目】（既存）
  消耗品費 / 工具器具備品 / ソフトウェア / 外注費 /
  広告宣伝費 / 旅費交通費 / 通信費 / 地代家賃 / 雑費

【未払金】
  ├── MFカード:未請求   ← Stage 1 で計上、Stage 2 で消込
  ├── MFカード:請求     ← Stage 2 で計上、Stage 3 で消込
  └── 従業員立替        ← 立替時に計上、精算振込で消込

【買掛金】
  └── （取引先名）      ← 請求書払い用
```

## 3. 支払方法別フロー

### 3.1 会社カード決済

**対象**: 従業員カード or 管理本部カードAでの購買・出張

#### 申請者の操作（Slackのみ）

```
田中さん:
  ① /purchase 52800 モニター Amazon（or /trip ...）
  ② 承認後、カード決済
  ③ 検収完了
  ④ Slackスレッドに証憑（納品書/領収書）を添付
  ⑤ 完了。以降の作業なし。
```

#### 購買システム（証憑添付時・自動）

```
⑤-a. Slackから証憑ファイルをダウンロード
⑤-b. Google Drive APIで証憑を保存
     → /購買証憑/2026/03/2026-03-15_52800_Amazon_PO-2026-0042_納品書.pdf
⑤-c. Gemini OCRで金額照合
⑤-d. POST /journals → MF会計Plusに仕訳ドラフト作成
     → status: "draft"
     → tags: ["PO-2026-0042"]
     → memo: "PO-2026-0042 Amazon モニター | 証憑: https://drive.google.com/file/d/xxx"
     → 借方: 消耗品費 ¥52,800
     → 貸方: 未払金(MFカード:未請求) ¥52,800
⑤-e. GAS更新（証憑=添付済, DriveURL, 仕訳ID, Stage=1）
⑤-f. Slack通知
```

#### 仕訳

```
Stage 1（証憑添付時・API自動）:
  借方: 消耗品費                  ¥52,800
  貸方: 未払金(MFカード:未請求)     ¥52,800
  → entered_by: api

Stage 2（カード確定値到着時・自動仕訳ルール）:
  借方: 未払金(MFカード:未請求)     ¥52,800
  貸方: 未払金(MFカード:請求)       ¥52,800
  → entered_by: none（MF会計Plusが自動登録）

Stage 3（銀行引落時・自動仕訳ルール）:
  借方: 未払金(MFカード:請求)       ¥52,800
  貸方: 普通預金                   ¥52,800
  → entered_by: none
```

  借方: 消耗品費                      ¥8,800
  貸方: 未払金(従業員立替)              ¥8,800
  → MF経費→MF会計Plus仕訳連携（証憑自動添付）

Stage 3（立替精算時・MF経費の精算機能）:
  借方: 未払金(従業員立替)              ¥8,800
  貸方: 普通預金                       ¥8,800

**対象**: 管理本部カードBでの月額サブスク（AWS, Slack, Google Workspace等）

## 4. カード明細マッチング

### 4.1 アーキテクチャ

```
MF会計Plus:
  自動仕訳ルール:
    従業員カード・管理本部カードAの全明細
      → 未払金(MFカード:未請求) / 未払金(MFカード:請求)
      → entered_by: none として仕訳登録

    管理本部カードBの明細
      → 費用科目 / 未払金(MFカード:請求)（加盟店名で科目判定）

購買システム（月次バッチ）:
  ① GET /journals?entered_by=none
     → 自動仕訳ルールで登録されたStage 2仕訳を全件取得
  ② GASから「会社カード + Stage 1済み + 未照合」の購買・出張レコードを取得
  ③ スコアリングでマッチング
  ④ 結果処理（後述）
```

### 4.2 マッチングの入力データ

```
左側: 購買・出張レコード（GAS）
  [{
    number: "PO-2026-0042",     // or "TR-2026-0015"
    type: "purchase",            // or "trip"
    amount: 52800,
    date: "2026-03-15",
    supplier: "Amazon",          // or "JR東海"
    applicant: "田中太郎",
    stage1JournalId: 12345
  }, ...]

右側: Stage 2仕訳（MF会計Plus API）
  [{
    journalId: 99001,
    date: "2026-03-15",
    amount: 52800,
    remark: "AMAZON.CO.JP"       // 加盟店名
  }, ...]
```

### 4.3 スコアリングアルゴリズム

```
totalScore = amountScore(50) + dateScore(30) + nameScore(20)

金額スコア（最大50点）:
  差額率 = |購買金額 - カード金額| / 購買金額
  一致(0%):     50点
  1%以内:       40点（税端数・為替微差）
  5%以内:       25点（送料追加等）
  10%以内:      10点（要確認）
  10%超:         0点

日付スコア（最大30点）:
  同日: 30 / 1日差: 25 / 2日差: 20 / 3日差: 15 / 4-7日: 5 / 8日超: 0

取引先名スコア（最大20点）:
  正規化後一致: 18 / 部分一致: 15 / 先頭3文字一致: 8 / 不一致: 0

判定:
  ≥80: 確信マッチ（自動処理）
  50-79: 候補マッチ（経理確認）
  <50: 未マッチ
```

### 4.4 取引先名の正規化

```typescript
function normalizeMerchantName(name: string): string {
  return name
    .toUpperCase()
    .replace(/\s+/g, "")
    .replace(/\.CO\.JP$/i, "")
    .replace(/\.COM$/i, "")
    .replace(/（.*?）/g, "")
    .replace(/\(.*?\)/g, "")
    .replace(/株式会社|㈱/g, "")
    .replace(/[ー－\-]/g, "");
}
```

### 4.5 加盟店名エイリアス辞書

```
| 購買台帳の取引先名 | カード明細の加盟店名パターン |
|------------------|-------------------------------|
| Amazon           | AMAZON.CO.JP, AMAZON DOWNLOADS |
| モノタロウ         | MONOTARO, モノタロウ |
| JR東海            | JR TOKAI, ジェイアール東海 |
| ANA              | ANA, 全日空, ALL NIPPON |
| じゃらん           | JALAN, リクルート |
```

管理画面で編集可能。マッチング実績から自動提案も可能（Phase 2）。

### 4.6 マッチング結果の処理

#### 確信マッチ（金額一致）

```
処理:
  → GAS更新: stage=2, stage2_journal_id=99001, card_matched=true
  → 追加仕訳は不要（Stage 2仕訳は自動仕訳ルールで登録済み）
```

#### 確信マッチ（差額あり）

```
例: 購買 ¥33,000 / カード ¥33,550（差額+¥550）

処理:
  → GAS更新: stage=2, stage2_journal_id=99002
  → 差額調整仕訳を自動作成:

  POST /journals:
    tags: ["PO-2026-0043", "Stage2-adj"]
    memo: "PO-2026-0043 差額調整 +¥550"
    branches:
      - debitor: 消耗品費 ¥550（Stage 1と同じ費用科目）
        creditor: 未払金(MFカード:未請求) ¥550

  → Stage 1の未請求残高が¥33,550に増え、Stage 2の振替額と一致
```

#### 未マッチのStage 2仕訳

```
従業員カード・管理本部カードAの明細で、購買・出張台帳にマッチしない
  → ★未申請アラート
  → Slack通知: "⚠️ 未申請のカード利用を検出: 3/18 ¥1,200 TAXI JAPAN"
  → 経理が確認:
    - 申請漏れ → 従業員に申請を依頼
    - 不正利用 → 対応
    - 正当な利用（購買システム外） → MF会計Plusで仕訳を修正
      （未払金:未請求/未払金:請求 → 費用科目/未払金:請求）
```

#### 未マッチの購買レコード

```
購買台帳にあるがStage 2仕訳にマッチしない
  → カード確定値がまだ出ていない（翌月繰越）
  → or キャンセル・返品
  → 翌月に再照合。2ヶ月連続未マッチはアラート
```

## 5. Google Drive証憑管理

### 5.1 フォルダ構成

```
📁 購買証憑/
├── 📁 2026/
│   ├── 📁 03/
│   │   ├── 2026-03-15_52800_Amazon_PO-2026-0042_納品書.pdf
│   │   ├── 2026-03-18_33000_モノタロウ_PO-2026-0043_請求書.pdf
│   │   ├── 2026-03-20_15400_JR東海_TR-2026-0015_領収書.pdf
│   │   └── ...
│   └── 📁 04/
└── 📁 _監査ログ/
    └── 証憑登録ログ_2026-03.csv
```

### 5.2 ファイル命名規則（電帳法検索要件対応）

```
{取引日}_{金額}_{取引先}_{番号}_{書類種別}.{拡張子}
例: 2026-03-15_52800_Amazon_PO-2026-0042_納品書.pdf
例: 2026-03-20_15400_JR東海_TR-2026-0015_領収書.pdf
```

### 5.3 電帳法対応

| 要件 | 対応方法 |
|------|---------|
| タイムスタンプ / 訂正削除履歴 | Google Vault（7年保持ルール） |
| 検索機能（日付・金額・取引先） | ファイル命名規則 + Drive検索 |
| 関連書類との相互関連性 | 仕訳メモにDriveリンク / Drive名にPO番号 |

### 5.4 リンク切れ防止

- Drive File IDベースのURL使用（フォルダ移動しても不変）
- 共有設定: 組織内リンク共有
- Vault保持ルールにより削除されても復元可能
- 月次整合性チェック: GAS全件 × Driveファイル存在確認

## 6. MF会計Plusの自動仕訳ルール設定

### 6.1 従業員カード・管理本部カードA（購買用）

```
全明細に適用:
  借方: 未払金(MFカード:未請求)
  貸方: 未払金(MFカード:請求)

理由:
  → 全利用が購買 or 出張のはず（購買システムで管理）
  → Stage 2振替仕訳として機能
  → 購買システムがGET /journals?entered_by=noneで取得しマッチング
  → 未マッチ = 未申請利用 → アラート

### 6.2 管理本部カードB（サブスク用）

```
加盟店名ベースのルール:
  "AWS" を含む       → 通信費 / 未払金(MFカード:請求)
  "SLACK" を含む     → 通信費 / 未払金(MFカード:請求)
  "GOOGLE" を含む    → 通信費 / 未払金(MFカード:請求)
  "ADOBE" を含む     → 消耗品費 / 未払金(MFカード:請求)

  借方: 未払金(MFカード:請求)
