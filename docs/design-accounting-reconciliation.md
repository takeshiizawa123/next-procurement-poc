# 会計照合設計: 3ステージ未払金管理モデル

**作成日**: 2026-03-26（改訂）
**方針**: 未払金を「未請求債務」「請求債務」に補助科目分割し、段階的に照合消込

---

## 1. 基本構造

### 1件の購買に対する3段階の仕訳

```
時系列:
  証憑添付 ──→ カード利用明細 ──→ 銀行引落
  (Stage 1)     (Stage 2)         (Stage 3)
```

| Stage | トリガー | 仕訳 | 金額根拠 |
|-------|---------|------|---------|
| 1 | 証憑添付 → [仕訳登録] | 費用 / 未払金(未請求) | 納品書・領収書 |
| 2 | カード利用明細到着 | 未払金(未請求) / 未払金(請求) | 利用明細 |
| 3 | 銀行引落 | 未払金(請求) / 普通預金 | 引落明細 |

### 未払金の補助科目構成

```
未払金
  └── MFビジネスカード
       ├── 未請求債務  ← Stage 1 で計上、Stage 2 で消込
       └── 請求債務    ← Stage 2 で計上、Stage 3 で消込
```

---

## 2. 各Stageの詳細

### Stage 1: 費用計上（証憑ベース）

**トリガー:** 管理本部が [仕訳登録] を押した時
**金額根拠:** 納品書・領収書の金額（OCR読取 or 手入力）

```
借方: 費用科目（消耗品費等）          ¥45,000
貸方: 未払金(MFカード:未請求債務)      ¥45,000
摘要: PR-0050 Amazon 会議用モニター
```

**ポイント:**
- カード利用明細がまだ来ていなくても、証憑があれば費用計上できる
- 未請求債務 = 「カード会社にはまだ請求されていないが、会社としては負債を認識」
- MF会計Plus API で `status: "draft"` で作成 → 管理本部が確認後に承認

---

### Stage 2: 利用明細との照合消込

**トリガー:** MFビジネスカードの利用明細がMF会計Plusに取り込まれた時
**処理:** 納品書金額（①）と利用明細金額（②）を照合

#### パターンA: 金額一致

```
借方: 未払金(MFカード:未請求債務)      ¥45,000
貸方: 未払金(MFカード:請求債務)        ¥45,000
摘要: PR-0050 未請求→請求 振替
```

#### パターンB: 利用明細のほうが少ない（返品・値引き等）

```
差額 = ① ¥45,000 - ② ¥43,000 = ¥2,000

借方: 未払金(MFカード:未請求債務)      ¥45,000
貸方: 未払金(MFカード:請求債務)        ¥43,000
貸方: 費用科目                        ¥2,000（差額戻し）
摘要: PR-0050 未請求→請求 振替（返品差額 ¥2,000）
```

#### パターンC: 利用明細のほうが多い（送料追加・為替差等）

```
差額 = ② ¥45,500 - ① ¥45,000 = ¥500

借方: 未払金(MFカード:未請求債務)      ¥45,000
借方: 費用科目                        ¥500（差額追加）
貸方: 未払金(MFカード:請求債務)        ¥45,500
摘要: PR-0050 未請求→請求 振替（為替差 ¥500）
```

#### パターンD: キャンセル（利用明細なし or マイナス）

```
借方: 未払金(MFカード:未請求債務)      ¥45,000
貸方: 費用科目                        ¥45,000
摘要: PR-0050 キャンセル（逆仕訳）
```

#### 為替について

```
海外EC購入の場合:
  ① 納品書: $300 → 購入時レート ¥150 = ¥45,000
  ② 速報値: $300 → 速報レート ¥150.5 = ¥45,150  ← ①とのズレ
  ③ 確定値: $300 → 確定レート ¥149.8 = ¥44,940  ← ②とのズレ

Stage 2は確定値（③）で実施するのが正確。
速報値（②）はあくまで参考（照合レポートに表示するが仕訳は切らない）。
```

**運用判断:** Stage 2を速報値で実施するか、確定値を待つか

| 方式 | タイミング | 精度 | 管理手間 |
|------|----------|------|---------|
| 速報時に実施 | 利用直後（即時） | △ 確定で差額再調整が必要 | 多い |
| **確定時に実施（推奨）** | 締日後 | ◎ 確定値で1回だけ | 少ない |

---

### Stage 3: 引落との照合消込

**トリガー:** 銀行口座に引落明細が取り込まれた時
**処理:** 引落額（④）とカード請求額（③の合計）を照合

```
借方: 未払金(MFカード:請求債務)     ¥XXX,XXX（当月請求額合計）
貸方: 普通預金                     ¥XXX,XXX
摘要: MFビジネスカード 3月分引落
```

**照合:** ④引落額 = Σ③確定値（当月分）

通常は一致する。不一致の場合はカード会社に問い合わせ。

---

## 3. 金額の流れ（まとめ）

```
           費用科目     未払金(未請求)   未払金(請求)    普通預金
           ────────    ─────────────   ────────────   ─────────
Stage 1:   +45,000     -45,000
Stage 2:               +45,000         -45,000
Stage 3:                                +45,000       -45,000

残高:      +45,000         0               0          -45,000
```

すべてのStageが完了すると未払金（未請求・請求とも）は**ゼロ**になる。

## 4. 照合タイミングと管理本部の運用

### 日次
```
09:00 日次サマリ確認
  → 「仕訳待ち」案件を処理（Stage 1 — 費用/未払金(未請求)）
```

### 週次
```
月曜 11:00 突合レポート（自動）
  → ① 納品書金額 ↔ ② カード速報値 を照合
  → 大きな差額 → アラート（返品・誤発注の可能性）
  → 速報値は参考情報のみ（仕訳は切らない）
```

### 月次（カード締日後）
```
確定明細が出たら:
  1. ① 納品書金額 ↔ ③ カード確定値 を照合
  2. 一致 → Stage 2 仕訳（未請求→請求 振替）
  3. 差額あり → 差額調整付きで Stage 2 仕訳
  4. キャンセル → 逆仕訳

引落日:
  5. ④ 引落明細 ↔ ③ 確定値合計 を照合
  6. 一致 → Stage 3 仕訳（請求債務→普通預金）
  7. 未払金(未請求) 残高 → 翌月繰越分 = 締日後の利用分
  8. 未払金(請求) 残高 → ゼロになるはず
```

## 5. 支払方法別の適用

| 支払方法 | Stage 1 | Stage 2 | Stage 3 | 備考 |
|---------|---------|---------|---------|------|
| **MFカード** | 費用/未払金(未請求) | 未請求→請求 振替 | 請求/普通預金 | フルモデル適用 |
| **請求書払い** | 費用/買掛金 | — | 買掛金/普通預金（振込時） | Stage 2不要（カードではない） |
| **立替（従業員）** | 費用/未払金(従業員) | — | 未払金(従業員)/普通預金 | Stage 2不要 |

## 6. MF会計Plusでの設定

### 必要な勘定科目・補助科目

```
未払金
  ├── MFビジネスカード:未請求債務   ← 新規作成
  ├── MFビジネスカード:請求債務     ← 新規作成
  └── （従業員名）                  ← 立替用（既存 or 新規）

買掛金
  └── （取引先名）                  ← 請求書払い用
```

### 自動仕訳ルール（MF会計Plus側）

```
カード利用明細が取込まれた時:
  → 「対象外」にする（Stage 2は購買システムからAPI仕訳で処理するため）
  → または自動仕訳ルールで「未請求→請求 振替」を自動化

銀行引落明細が取込まれた時:
  → 借方: 未払金(MFカード:請求債務)
  → 貸方: 普通預金
  → 自動仕訳ルールで設定可能
```

## 7. 購買システム側の実装

### Stage 1（既存の改修）

`buildJournalFromPurchase` の貸方を変更:

```
現在: 貸方: 未払金
変更: 貸方: 未払金(補助:MFカード:未請求債務)
```

### Stage 2（新規）

月次バッチ or 手動操作で:
1. GASから「仕訳済み + カード確定値あり + Stage 2未処理」の案件を取得
2. ① ↔ ③ を照合
3. 差額に応じた振替仕訳をMF会計Plus APIで作成
4. GASのステータスを更新

### Stage 3（MF会計Plus側で処理）

銀行引落は購買システムの範囲外。MF会計Plusの自動仕訳ルールで対応。

---

## 8. 差額の許容範囲

| 差額 | 処理 | 備考 |
|------|------|------|
| 0円 | Stage 2を単純振替 | |
| ±1〜10円 | 自動で差額調整仕訳 | 税端数として処理 |
| ±11〜500円 | アラート + 管理本部確認後に調整 | 送料差・軽微な変更 |
| ±501円〜 | アラート + 原因調査必須 | 返品・キャンセル・為替 |
