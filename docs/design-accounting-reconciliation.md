# 会計照合設計: 4段階の金額と突合モデル

**作成日**: 2026-03-26
**課題**: 購入時〜引落時で金額がズレる現実に対応する会計処理の設計

---

## 1. 4つの金額とその発生源

1件の購買に対して、最大4種類の金額が存在する:

```
時系列:
  購入 ──→ カード速報 ──→ カード確定 ──→ 銀行引落

① 納品書金額         ... 購買システム（証憑OCR）
② カード速報値       ... MFビジネスカード利用明細（即時〜数日）
③ カード確定値       ... MFビジネスカード確定明細（締日後）
④ 引落額            ... 銀行口座からの引き落とし（支払日）
```

| # | 金額 | ソース | タイミング | 精度 |
|---|------|--------|----------|------|
| ① | 納品書/領収書の金額 | 証憑OCR or 手入力 | 検収〜証憑添付時 | 確定（税込） |
| ② | カード利用速報値 | MFビジネスカード明細 | 決済直後〜数日 | **暫定**（為替未確定、返品前） |
| ③ | カード確定値 | MFビジネスカード確定明細 | 締日後 | **確定**（為替確定、返品反映済み） |
| ④ | 引落額 | 銀行口座明細 | 支払日（翌月25日等） | 確定（③の合計） |

### ズレが発生するケース

| ケース | ①と②のズレ | ②と③のズレ | 具体例 |
|--------|-----------|-----------|--------|
| 為替変動 | あり | あり | 海外EC（ドル建て→円換算レートが変動） |
| 部分キャンセル | なし | あり | 5個注文→3個に変更（確定時に減額） |
| 返品 | なし | あり | 購入後に返品→確定明細でマイナス |
| ポイント値引き | あり | なし | ポイント利用分が差し引かれている |
| 税端数 | あり | なし | 納品書の税計算と実決済額の1円差 |
| 送料変更 | あり | なし | 注文時と発送時で送料が変わった |

## 2. 会計処理モデル

### Stage 1: 購入時（証憑添付 → API仕訳）

**トリガー:** 管理本部が[仕訳登録]を押した時
**金額根拠:** ① 納品書金額（OCR or 手入力）

```
借方: 費用科目（消耗品費等）  ¥45,000  ← ①納品書金額
貸方: 未払金                  ¥45,000
摘要: PR-0050 Amazon 会議用モニター
```

**この時点で:**
- ②速報値はまだ来ていない or 来ているが仕訳には使わない
- ①が「正」の金額（取引の実態を反映）

### Stage 2: カード速報値との照合（週次突合バッチ）

**トリガー:** 週次Cron（カード明細突合）
**処理:** ①納品書金額 と ②速報値 を照合

```
照合結果:
  一致（許容差 ±500円以内）→ マッチ。経過観察。
  不一致 → アラート通知。管理本部が確認。
  未マッチ → 未申請購入 or タイミングずれ。

※ この段階では仕訳は作らない。照合レポートのみ。
```

**MF会計Plus上のカード利用明細は「対象外」にする:**
- API仕訳(Stage 1)で費用/未払金は計上済み
- カード利用明細から二重に仕訳を作らない

### Stage 3: カード確定値との照合（月次・締日後）

**トリガー:** 月次（カード締日後、確定明細が出た時点）
**処理:** ①納品書金額 と ③確定値 を照合

```
パターンA: 一致 → 何もしない。Stage 4を待つ。

パターンB: 差額あり（為替・返品等）
  ③確定値が①より少ない場合（返品・値引き等）:
    借方: 未払金      ¥2,000（差額）
    貸方: 費用科目    ¥2,000
    摘要: PR-0050 カード確定差額調整（返品）

  ③確定値が①より多い場合（為替差損等）:
    借方: 費用科目    ¥500（差額）
    貸方: 未払金      ¥500
    摘要: PR-0050 カード確定差額調整（為替）

パターンC: キャンセル（③がゼロ or マイナス）
  借方: 未払金        ¥45,000
  貸方: 費用科目      ¥45,000
  摘要: PR-0050 全額キャンセル（逆仕訳）
```

### Stage 4: 銀行引落（支払日）

**トリガー:** 銀行口座連携で引落明細が取込まれた時
**処理:** MF会計Plusの銀行明細から仕訳

```
借方: 未払金      ¥XX,XXX（カード請求額 = ③の合計）
貸方: 普通預金    ¥XX,XXX
摘要: MFビジネスカード 3月分引落

※ ④引落額 = 当月の③確定値の合計
※ 個別のPO番号との紐付けは不要（未払金の総額で一致すればOK）
```

## 3. 突合マトリクス

```
            ① 納品書    ② 速報     ③ 確定     ④ 引落
① 納品書    ---        週次照合    月次照合    ---
② 速報      週次照合   ---        締日照合    ---
③ 確定      月次照合   締日照合   ---        引落照合
④ 引落      ---        ---        引落照合   ---
```

| 照合ペア | タイミング | 方法 | 差異時の処理 |
|---------|----------|------|------------|
| ①↔② | 週次 | 購買システム突合バッチ | アラート通知のみ |
| ①↔③ | 月次 | 手動 or 月次バッチ | **差額調整仕訳を作成** |
| ③↔④ | 引落時 | MF会計Plusの銀行明細照合 | 通常は一致（不一致は銀行問合せ） |

## 4. 購買システムでの管理項目

### GASスプレッドシートに追加が必要な列

| 列名 | 内容 | 入力タイミング |
|------|------|-------------|
| 納品書金額 | ①の金額（OCR or 手入力） | 証憑添付時 |
| カード速報額 | ②の金額 | 週次突合時（自動） |
| カード確定額 | ③の金額 | 月次（手動 or API） |
| 差額 | ③ - ① | 自動計算 |
| 差額処理ステータス | 未処理/調整済み/不要 | 管理本部が更新 |
| MF仕訳ID（購入時） | Stage 1の仕訳ID | 仕訳登録時（自動） |
| MF仕訳ID（差額調整） | Stage 3の仕訳ID | 差額調整時（自動） |

## 5. 運用フロー（管理本部）

### 日次
```
09:00 日次サマリ確認
  → 「仕訳待ち」案件を処理（Stage 1 — API仕訳作成）
  → MF会計Plusで「カード利用明細」を「対象外」にする
```

### 週次
```
月曜 11:00 カード明細突合レポート（自動）
  → ①↔② の照合結果を確認
  → 差額大のものは原因調査（返品？為替？）
  → MF会計Plusの「通帳・カード他」で対象外処理の漏れがないか確認
```

### 月次（締日後）
```
カード確定明細が出たら:
  1. ①↔③ の照合を実施
  2. 差額がある案件を特定
  3. 差額調整仕訳を作成（Stage 3）
  4. GASの差額処理ステータスを更新

引落日:
  5. 銀行明細から「未払金/普通預金」仕訳を作成（Stage 4）
  6. 未払金残高がゼロになることを確認
```

## 6. 差額の許容範囲と自動処理

| 差額 | 処理 |
|------|------|
| 0円 | 何もしない |
| ±1〜10円 | **自動で差額調整仕訳**（端数処理として） |
| ±11〜500円 | アラート + 管理本部確認後に調整仕訳 |
| ±501円〜 | アラート + 原因調査必須（返品・為替・誤発注等） |

## 7. 請求書払い・立替の場合

### 請求書払い

```
Stage 1: 費用/買掛金（請求書添付時）
Stage 3: 不要（カードではないので速報/確定の概念がない）
Stage 4: 買掛金/普通預金（振込時）

差額: 請求書金額と振込額が一致するか確認（通常は一致）
```

### 立替

```
Stage 1: 費用/未払金(従業員)（レシート添付時）
Stage 3: 不要（カード利用ではないので）
Stage 4: 未払金(従業員)/普通預金（精算振込時）

差額: レシート金額と精算額は一致（差額なし）
```

## 8. 実装への影響

### 必要な追加開発

| # | 作業 | 内容 | 優先度 |
|---|------|------|--------|
| 1 | GAS列追加 | カード速報額・確定額・差額・差額処理ステータス | 高 |
| 2 | 週次突合バッチ改修 | ①↔②照合結果をGASに書き戻し | 高 |
| 3 | 月次差額照合機能 | ①↔③照合 + 差額調整仕訳API | 中 |
| 4 | カード利用明細の対象外処理 | MF会計PlusのAPI or 運用手順書 | 中 |
| 5 | 差額自動調整（±10円以内） | 端数差額の自動仕訳 | 低 |
| 6 | 引落照合 | ③合計↔④の一致確認 | 低 |

### 判断が必要な事項

1. **カード利用明細の「対象外」処理** — API化できるか、手動運用にするか
2. **確定値の取得方法** — MFビジネスカードAPIがあるか、手動入力か
3. **差額調整の承認フロー** — 自動仕訳にするか、管理本部の確認を挟むか
4. **未払金の補助科目** — カード会社名で管理するか、PO番号単位で管理するか
