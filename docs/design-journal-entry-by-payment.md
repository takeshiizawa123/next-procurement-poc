## 1. 全体像

### 支払方法と証憑・仕訳の経路

```
                          証憑保管        仕訳作成           精算
                          ──────        ──────            ────
会社カード ──→ 案G ──→ Google Drive ──→ POST /journals ──→ カード引落（自動）
請求書払い ──→ 案G ──→ Google Drive ──→ POST /journals ──→ 振込（経理が実行）
従業員立替 ──→ 案C ──→ MF経費       ──→ MF経費→会計連携 ──→ MF経費で精算振込
```

### 分岐ロジック

```typescript
// 証憑添付時の自動分岐
if (paymentMethod === "立替") {
  // 案C: MF経費経由
  await uploadReceiptToMfExpense(file);    // 証憑UP + OCR明細生成
  await createExTransactionForPurchase();  // 明細補完（PO番号・科目）
  // → 経理がMF経費UIで申請・承認 → MF会計Plus仕訳連携
} else {
  // 案G: Drive + API仕訳（会社カード / 請求書払い）
  await uploadToDrive(file);               // Drive保存
  await createJournal(journalRequest);     // MF会計Plus仕訳ドラフト
  // → 経理がMF会計Plusで仕訳承認
}
```

### Stage 2: カード確定値との照合振替（月次・締日後）

**トリガー**: カード利用明細の確定値が出た後（月次バッチ or 手動）
**処理**: 納品書金額（Stage 1）と カード確定値 を照合

```

#### パターンB: 差額あり（送料追加・為替差・値引き等）

```

仕訳:
  借方: 未払金(MFカード:未請求)     ¥52,800
  借方: 消耗品費                   ¥500      ← 差額追加
  貸方: 未払金(MFカード:請求)       ¥53,300
  摘要: PO-2026-0042 未請求→請求 振替（差額+¥500）

```

#### パターンC: キャンセル

## 5. 従業員立替

### Stage 1: 費用計上（MF経費経由・月次）

**トリガー**: 証憑添付 → MF経費にupload_receipt → 経理が月次で申請・承認
**経路**: 案C（MF経費 → MF会計Plus仕訳連携）

```
購買システム（証憑添付時・即時）:
  ① upload_receipt → MF経費に証憑+OCR明細
  ② ex_transaction → 明細を補完
     → 経費科目: 消耗品費
     → 備考: "【立替】PO-2026-0046 ヨドバシ USBケーブル / 立替者: 田中太郎"
     → 金額: ¥8,800

経理（月次・MF経費UI）:
  ③ 立替の購買明細を選択 → 経費申請作成
  ④ 承認
  ⑤ MF会計Plus仕訳連携 → 仕訳が自動作成（証憑添付済み）

MF会計Plusに作成される仕訳:
  借方: 消耗品費                      ¥8,800
  貸方: 未払金(従業員立替: 田中太郎)    ¥8,800
  摘要: PO-2026-0046 ヨドバシ USBケーブル
  証憑: 領収書PDF ← MF経費から自動添付

**注意**: 仕訳作成タイミングが会社カード/請求書と異なる
- 会社カード/請求書: 証憑添付時に即座にドラフト作成

## 8. mf-accounting.ts の修正箇所

### 現在の貸方ロジック（要修正）

```typescript
// 現在（L194-197）
const isCard = paymentMethod.includes("カード");
const creditAccountName = isCard ? "未払金" : "買掛金";
```

### 修正後

```typescript
// 支払方法別の貸方科目を解決
function resolveCreditAccount(paymentMethod: string): {
  accountName: string;
  subAccountName?: string;
} {
  if (paymentMethod.includes("カード")) {
    return {
      accountName: "未払金",
      subAccountName: "MFビジネスカード:未請求",
    };
  }
  if (paymentMethod === "立替") {
    // 立替分は案C経由のため、この関数は呼ばれない
    // （念のため定義）
    return {
      accountName: "未払金",
      subAccountName: undefined, // 従業員名は別途設定
    };
  }
  // 請求書払い
  return {
    accountName: "買掛金",
    subAccountName: undefined, // 取引先名は別途設定
  };
}
```

### 分岐ポイント（events/route.ts）
