# 案B詳細設計: MF会計Plus直接仕訳（MF経費バイパス）

**作成日**: 2026-03-26
**目的**: 申請者の証憑提出を1回（Slackスレッド or マイページ）で完結させる

---

## 1. 基本コンセプト

### Before（現状の想定）
```
申請者: Slack証憑添付 → MF経費で証憑再添付 → MF経費で経費申請提出
                          ↑ 二重作業        ↑ 三重作業
```

### After（案B）
```
申請者: Slack証憑添付（1回だけ）
  ↓ 自動
購買システム:
  ├── OCR金額照合
  ├── GAS「添付済」更新
  └── 証憑ファイルを一時保持
  ↓
管理本部: [仕訳登録] ボタン（1クリック）
  ↓ 自動
MF会計Plus API:
  ├── 仕訳ドラフト作成（借方:費用 / 貸方:未払金 or 買掛金）
  ├── 証憑PDFを仕訳に添付（電帳法対応）
  └── タグにPO番号を付与
  ↓
MF会計Plus内:
  └── カード明細（自動取込）と仕訳が自動マッチ
```

**申請者の操作: Slackに証憑を1回添付するだけ。MF経費を開く必要なし。**

### パターンC: 請求書払い


**ポイント:** 貸方が「未払金」の補助科目を「従業員名」にする。精算は給与振込や個別振込で処理。

## 3. MF会計Plusとカード明細の突合

### MFビジネスカード → MF会計Plusの連携

```
MFビジネスカードで決済
  ↓（自動・数日後）
MF会計Plus「自動で仕訳」画面にカード明細が表示
  ↓
2つの処理方法:

方法1（推奨）: 購買システムからAPI仕訳 → カード明細と手動マッチ
  - API仕訳がドラフトとして作成される
  - 管理本部がMF会計Plus上で明細と仕訳を紐付け
  - タグ(PO番号)で検索して対応づけ

方法2: カード明細から直接仕訳 + 購買システムの仕訳をスキップ
  - MF会計Plusの「自動で仕訳」からカード明細を選んで仕訳
  - 購買システム側は「仕訳済み」にステータス更新のみ
  - ★ この場合、購買システムからの仕訳登録は不要
```

### 推奨運用: 方法1（API仕訳 + 手動マッチ）

理由:
- 購買システムのOCR照合結果を仕訳に反映できる
- PO番号タグで購買台帳との追跡が容易
- 証憑PDFがMF会計Plusに自動添付される（電帳法対応）

## 4. MF経費の役割変更

### 案B導入後のMF経費の位置づけ

| 用途 | MF経費を使う？ | 理由 |
|------|-------------|------|
| **購買申請（カード/請求書）** | **使わない** | 購買システム → MF会計Plus直接 |
| **出張費（交通・宿泊）** | **使う** | /trip + じゃらんCSV → MF経費 → MF会計Plus |
| **接待交際費** | **使う** | 少額・即時精算 → MF経費が適切 |
| **日常経費（タクシー・駐車場等）** | **使う** | 購買申請には馴染まない経費 |

**整理:**
- 購買（物品・サービス購入）→ **購買システム → MF会計Plus**
- 経費（出張・接待・日常）→ **MF経費 → MF会計Plus**

MF経費は廃止ではなく、**購買以外の経費に特化**する。

## 5. 必要な実装変更

### 5.1 証憑のMF会計Plus添付（新規）

現状の `/api/mf/journal` は仕訳作成のみ。証憑PDFの添付が未実装。

```
MF会計Plus API:
  POST /journals → 仕訳作成（実装済み）
  POST /journals/{id}/attachments → 証憑添付（要実装）
```

**実装内容:**
- 仕訳作成時に、Slackから証憑ファイルをダウンロード
- MF会計Plus APIで仕訳に証憑PDFを添付
- 電帳法タイムスタンプ要件への対応

### 5.2 貸方科目の分岐強化

現状:
```typescript
const isCard = paymentMethod.includes("カード");
const creditAccountName = isCard ? "未払金" : "買掛金";
```

変更後:
```typescript
// 立替の場合は「未払金」の補助科目を従業員名にする
const isCard = paymentMethod.includes("カード");
const isReimbursement = requestType === "購入済";
const creditAccountName = isReimbursement ? "未払金" : isCard ? "未払金" : "買掛金";
const creditSubAccount = isReimbursement ? applicantName : isCard ? "MFビジネスカード" : supplierName;
```

### 5.3 MF経費への自動転送を条件付きに

購買申請 → MF経費転送は不要に。出張(/trip)のみ転送。

```
証憑添付時:
  if (購買申請) → MF経費転送しない（MF会計Plus直接）
  if (出張申請) → MF経費転送する（既存フロー維持）
```

### 5.4 仕訳登録ボタンの強化

管理本部が[仕訳登録]を押した時:
1. 仕訳作成（現状通り）
2. **証憑PDFをMF会計Plusに添付**（新規）
3. GASステータス更新（現状通り）
4. Slack通知（現状通り）

## 8. 前提条件と確認事項

### MF会計Plus側で必要な設定

| 設定 | 内容 | 確認状況 |
|------|------|---------|
| API仕訳作成の権限 | OAuth scope に `journal.write` | Sprint 3で実装済み |
| 仕訳への添付ファイルAPI | `POST /journals/{id}/attachments` の存在確認 | **要確認** |
| カード明細の自動取込 | MFビジネスカード → MF会計Plus連携 | **要確認** |
| 未払金の補助科目 | 「MFビジネスカード」補助科目の存在 | **要確認** |
| 買掛金の補助科目 | 取引先名で自動作成 or 手動作成 | **要確認** |

### 確認が必要な事項

1. **MF会計Plusの仕訳添付API** — 証憑PDFを仕訳に添付するAPIエンドポイントがあるか
2. **MFビジネスカード → MF会計Plus** の自動連携が有効化されているか
3. **電帳法対応** — API経由の証憑添付がタイムスタンプ要件を満たすか
4. **立替精算の振込** — MF会計Plus上で振込データ作成が可能か、それとも別途処理か
