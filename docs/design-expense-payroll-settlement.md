# 設計書: 立替精算と給与連携

**作成日**: 2026-04-18
**ステータス**: 設計（実装未着手、要件固め待ち）

## 背景

現在の立替精算フロー（expenseFlow）は以下が実装済み:
- `/expense/new` で申請（支払方法=立替）
- 承認→発注→検収ステップをスキップ
- 証憑OCR→金額照合→仕訳生成
- MF会計Plusに費用計上（借方: 費用 / 貸方: 未払金）

**欠けている部分**: 未払金の **従業員への振込処理**。現状は経理が手作業でMF経費から給与振込システムへデータを手動転記している。

## 目的

立替精算→給与合算→振込データ生成を自動化し、手作業を排除する。

## 現状のデータフロー

```
[従業員: /expense/new 立替申請]
  ↓
[purchase_requests テーブルに記録]
  ↓
[証憑OCR + 金額照合]
  ↓
[MF会計Plus 仕訳: 借方費用/貸方未払金]
  ↓
[ここまで自動] ————— [ここから手動] ————→
                          ↓
                  [経理: 各従業員の未払金を集計]
                          ↓
                  [給与計算システムに手動入力]
                          ↓
                  [月次給与と合算して振込]
```

## 提案する自動化

### Phase 1: 給与連携データ出力（実装容易）

**ゴール**: 未払金一覧をCSVで出力、給与担当者が給与計算システムに取り込む

- **新規API**: `/api/admin/expense/payroll-csv?month=YYYY-MM`
  - 当月の `purchase_requests` から `payment_method="立替"` かつ `voucher_status="仕訳済"` を抽出
  - 従業員別に集計
  - CSV形式: `従業員ID, 従業員名, 対象月, 合計金額, 申請番号リスト`
- **新規UI**: `/admin/expense/payroll` ページで月を選択→CSVダウンロード

**工数**: 約2時間

### Phase 2: 給与計算システムAPI直接連携（要調整）

**ゴール**: 給与計算システムのAPIに直接データ送信

前提:
- 給与計算システムがAPI提供している（MF給与、freee、マネーフォワードクラウド給与、等）
- API認証情報を保持

フロー:
```
[月次cron: 毎月給与締め日（例: 25日）]
  → expense-payroll-export cron
  → 対象月の立替精算仕訳を集計
  → 給与計算システムAPIに POST （項目: 立替精算）
  → 振込済み反映をpurchase_requestsに記録
```

**工数**: API選定 + 認証セットアップ + 実装 = 約6時間

### Phase 3: 仕訳の完全自動化（理想）

支払済み後の仕訳:
- 借方: 未払金
- 貸方: 給与・普通預金
- MF会計Plusに自動登録

振込照合:
- 銀行口座明細取得（MF会計の銀行連携経由）
- 給与振込明細との突合
- 不一致は警告

**工数**: 約8時間（振込照合ロジック含む）

## 必要な意思決定（要ユーザー確認）

1. **給与計算システムは何を使っているか**？
   - MF給与 / freee / マネーフォワード給与 / 弥生給与 / 独自
   - → Phase 2実装時にAPI接続方法が決まる

2. **立替精算の振込タイミング**は？
   - a) 月次給与と同時（締め日+月末払いなど）
   - b) 独立振込（都度）
   - c) 一定金額以上は即時振込

3. **税務処理**
   - 立替 ≠ 給与のため源泉徴収対象外
   - 給与計算システム側で「立替精算」として非課税扱いされるか確認

4. **承認プロセス**
   - Phase 1: CSV出力だけなら承認不要
   - Phase 2以降: 自動振込指図は経理承認を挟むべき

## 実装優先度

**短期（1週間以内）**: Phase 1（CSV出力）のみ実装
- 最小工数で最大の時間短縮効果
- 給与計算システムの仕様に依存しない

**中期（2-4週間）**: 給与計算システム確定後にPhase 2を実装

**長期**: Phase 3は会計責任者の承認後に検討
